My Approach

Lessons I Learned the Hard Way

A few things I keep coming back to after every project.

🧱

Test It at 10x

My wildfire pipeline worked great on a sample. Then I ran it on 150K records and everything broke. Now I always ask "what happens when this gets 10 times bigger?" before I write the first line.

🔍

Dirty Data Isn't a Bug

At Tech Mahindra the raw data was a mess. Missing fields, wrong formats, duplicates everywhere. I used to see that as a problem to get past. Now I see it as the first layer of analysis. The mess tells you something.

🗣️

If Nobody Uses It, It Failed

I built a dashboard that was technically solid and nobody opened it for two weeks. That stung. Now I sit with the people who'll use the thing before I start building it. Saves everyone time.

Machine Learning

Handwritten Digit Classification: When Is Deep Learning Actually Worth It?

Everyone in the class used CNNs and called it a day. I wanted to actually understand the tradeoff, so I built the full pipeline with four different models: Logistic Regression, Random Forest, MLP, and CNN. The CNN got 99% accuracy, sure, but the more useful finding came from the error analysis. Classical models kept confusing 4s and 9s, and 3s and 8s because they couldn't learn spatial relationships the way convolutions do. The confusion matrices told a clearer story than any single accuracy number. I walked away from this project with a rule I still follow: always look at where the model fails before you celebrate where it succeeds.

Big Data + ML

Wildfire Risk Prediction: The Problem With Rare Events

Here's the thing about predicting wildfires: they almost never happen in your dataset. Out of 150,000+ records, the positive class was tiny. A naive model could predict "no wildfire" every time and still score well. So the real work was in the data engineering. I generated 60,000+ synthetic samples using oversampling techniques, then hand-engineered 10+ features from raw environmental data (things like terrain slope, vegetation dryness indices, and wind speed patterns). I built the whole pipeline in PySpark on Databricks because my laptop literally could not handle the joins. Random Forest got 91% accuracy and 0.73 AUC-ROC. Not perfect, but for a problem where a false negative could mean an undetected fire, I spent more time optimizing recall than overall accuracy.
